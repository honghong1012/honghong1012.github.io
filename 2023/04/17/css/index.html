Box model

basic concept in web design -> how the elements are structured and interact
every html element -> rectangular box -> contains content,padding,border,margin
content -> text/media
padding -> space between border and content
border -> line surround padding and content
margin -> space between boarder and neighbor element

Box-sizing

property specify how total width/height of element is calculated
taking into accounts -> content padding border margin
there are 2 values for this property

  1. content-box -> default value + specify width/height of the element only includes content
  2. border-box -> includes content padding border but without margin

Static / relative / absolute / fixed / sticky position

  • static: default unpositioned
  • relative: original position as a reference -> other ele won’t take its place
  • absolute: use the closet positioned element as a reference, otherwise, use body
  • fixed: use viewport as reference
  • sticky: combination of relative and fixed -> relative on default -> fixed when scroll out of the web page

z-index

property to decide the stacking order -> only works for positioned elements

visibility: hidden / display: none

visibility: hidden;
hides an element from view, but the element still takes up space on the web page. The element is not visible, but its layout and size are still preserved, so other elements on the page are positioned as if it were still visible.

display: none; completely removes an element from the web page, including its layout and size.

px / em / rem / vh vw / %

Px, em, rem, vh, vw, and % are all units of measurement in CSS

  • Px (pixels) -> absolute unit of measurement -> a single dot on a screen. It’s commonly used for precise measurements and when creating fixed-width layouts.
  • Em -> relative unit -> based on the font size of the parent element.
  • Rem (root em) relative unit -> based on the font size of the root element (typically the element).
  • Vh (viewport height) and vw (viewport width) -> relative units of measurement -> based on the height and width of the browser viewport, respectively.
  • % (percentages) -> relative unit -> based on the size of the parent element.

Flex / grid

  • Axis: Flexbox is designed for one-dimensional layouts, either in a row or a column, while CSS Grid is designed for two-dimensional layouts with rows and columns.
  • Item placement: Flexbox provides a way to align and distribute items along the main axis and cross axis, while CSS Grid allows for more fine-grained placement of items in a grid, with control over row and column placement, spanning, and alignment.
  • Browser support: CSS Grid is supported by most modern browsers, including all major desktop and mobile browsers. Flexbox has wider support, but some older browsers may not support all of its features.

When to choose Flexbox:
For simpler layouts that only require one-dimensional alignment of items, such as navigation menus, card layouts, or simple lists.

When to choose grid:
For more complex layouts that require precise placement of items in rows and columns, such as magazine-style layouts, image galleries, or dashboard designs.

Block / Inline / inline-block

block

  • respect all of those
  • force a line break after the block element
  • breaks the flow

inline:

  • respect left & right margins and padding, but not top & bottom
  • cannot have a width and height set
  • margin and padding will push other elements horizontally not vertically
  • allow other elements to sit to their left and right.
  • elements do not break the flow

inline-block:

  • allow other elements to sit to their left and right
  • respect top & bottom margins and padding
  • respect height and width

resetting / normalizing

Resetting - Resetting is meant to strip all default browser styling on elements. For e.g. margins, paddings, font-sizes of all elements are reset to be the same. You will have to redeclare styling for common typographic elements.

Normalizing - Normalizing preserves useful default styles rather than “unstyling” everything. It also corrects bugs for common browser dependencies.

specificity

In CSS, specificity is a way of determining which style rules should be applied to an element when multiple rules could apply to the same element.

It is used to resolve conflicts between different style rules that target the same element, by giving priority to the most specific rule.

Specificity is calculated using a system of values assigned to different parts of a CSS selector. The selector with the highest specificity will override any conflicting rules with lower specificity.

The specificity values are calculated as follows:

ID selectors: 100
Class selectors, attribute selectors, and pseudo-classes: 10
Type selectors and pseudo-elements: 1
Universal selector (*), combinators (+, >, ~), and the negation pseudo-class (:not()) have no specificity value.
To calculate the specificity of a selector, you add up the specificity values of each of its parts. For example, the selector “body .wrapper div#content p” would have a specificity of 111 (1 for the type selector p, 10 for the ID selector #content, 10 for the class selector .wrapper, and 100 for the type selector div inside the ID selector).

When multiple rules have the same specificity, the last one declared in the stylesheet takes precedence.

What does a DOCTYPE do?

DOCTYPE is an abbreviation for DOCument TYPE.
A DOCTYPE is always associated to a DTD for Document Type Definition.

A DTD defines how documents of a certain type should be structured (i.e. a button can contain a span but not a div), whereas a DOCTYPE declares what DTD a document supposedly respects (i.e. this document respects the HTML DTD).

For webpages, the DOCTYPE declaration is required. It is used to tell user agents what version of the HTML specifications your document respects.
Once a user agent has recognized a correct DOCTYPE, it will trigger the no-quirks mode matching this DOCTYPE for reading the document.
If a user agent doesn’t recognize a correct DOCTYPE, it will trigger the quirks mode.

alt

The alt attribute provides alternative information for an image if a user cannot view it. The alt attribute should be used to describe any images except those which only serve a decorative purposes, in which case it should be left empty.

b and strong

The b tag is a presentational tag that simply makes text bold.

The strong tag, on the other hand, is a semantic tag that is used to indicate that the enclosed text is of particular importance or significance.

b should be used for presentational purposes, such as making text bold for aesthetic reasons, while should be used to indicate the importance or significance of text for semantic and accessibility purposes.

Responsive design

Responsive design is an approach to web design and development that aims to create websites and applications that provide an optimal viewing and interaction experience across a wide range of devices and screen sizes, from desktop computers to mobile phones.

use @media to query the user’s device width or height to override the styles

Mobile first design

Mobile first design is an approach to web design that prioritizes the design and development of the mobile version of a website or application before creating the desktop version. This means that designers and developers focus on designing for smaller screens first, and then gradually add more complex features and layout elements as the screen size increases.

The mobile first approach is driven by the increasing use of mobile devices to access the internet and the need for websites and applications to be optimized for smaller screens.
