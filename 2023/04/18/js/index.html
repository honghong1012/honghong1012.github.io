Value type / Reference type

Value types, also known as primitive types, include strings, numbers, booleans, null, and undefined.

  • value type is assigned to a variable or passed as an argument to a function -> a copy of the value is created and stored in a new location in memory
  • Any changes made to the variable or parameter do not affect the original value.

Reference types, on the other hand, include objects, arrays, and functions.

  • When a reference type is assigned to a variable or passed as an argument to a function -> a reference to the original object is created and stored in memory
  • This means that changes made to the variable or parameter affect the original object, since they are both pointing to the same location in memory.

To compare value types, you can simply use the === operator, which checks for strict equality.
To compare reference types, however, you need to be careful. The === operator checks for equality of reference, not equality of content. loop + JSON.stringify() to compare the content

Shallow copy / deep copy

techniques used to create copies of objects or arrays

Shallow copy

  • Create a new obj/array and copy the properties of ele of the original one to the new one
  • However if original one contains nested, the references to those would be copied. -> changes to nested would affect
  • Useful when only need to modify the top-level properties or ele of an obj/array or create light-weight copy of large data structure
  • Method: Object.assign(), spread operator(…)

Deep copy

  • Create a new obj/array and recursively copy all the properties/eles, including nested ones. -> changes would not affect original
  • Slower and more memory-intensive
  • Method: Lodash library, JSON.parse(JSON.stringify())

var / let / const

variable can be declared using these keywords

var

  • oldest way to declare
  • function scope -> variable is accessible within the function it is defined in, as well as the any nested function -> outside the function becomes a global variable
  • re-declared and updated within the same scope
  • variable hoisting -> declarations are moved to top

let / const

  • introduced in ES6
  • block scope -> block is a section of code surrounded by curly braces(if/else, loop)
  • let can be updated but not re-declared within same scope
  • const cannot be reassigned or re-declared

closure

A closure is an inner function that has access to the outer function’s variables.
It has three scope chains: access to its own scope -> outer function’s variables -> global variables

Event loop

  • key concept in the JS runtime environment -> enables asynchronous programming.
  • In JS, code is executed in a single-threaded environment, which means that only one task can be processed at a time. -> However, JavaScript also supports non-blocking I/O operations, which allow for asynchronous code execution.
  • To manage asynchronous operations, JS uses the event loop.
  • The event loop is a continuous process that monitors the call stack and the task queue. Whenever the call stack is empty, the event loop looks at the task queue and adds any pending tasks to the call stack. This way, asynchronous tasks can be executed when they are ready, without blocking the main thread.
  • micro(promise callback) > macro(setTimeout/setInterval)

Basic workflow

  • When a task is added to the task queue, it is marked as “pending”.
  • When the call stack is empty, the event loop checks the task queue for any pending tasks.
  • If there is a pending task, it is added to the call stack and executed.
  • Once the task is complete, the call stack is empty again and the event loop checks for any new pending tasks.

Promise

  • container -> have the result of the future things
  • provides cleaner and intuitive way to deal with asynchronous tasks, avoid callback hell
  • two states: pending and settled, settled includes fulfilled + rejected
  • uses method “then” and “catch” to deal with all kinds of asynchronous operation

Async / why

Event loop. We want non-blocking and avoid DOM render conflicting

async / await difference with Promise

same thing -> syntax sugar that allows to write async code like sync code.

“this” in js

“this” means the environment object where the function is running.
With “Function” keyword:

  • “this” in constructor function. -> the function called with “new” -> this refers the instance
  • “this” in an object method function -> refers to who call the function (the thing before the “.”)
  • “this” in a plain function -> window
  • “this” in event handler -> the element fires the event

Arrow function
“this” points to closest “this” when create the function, it is lexical scope
Arrow function cannot be the constructor function

bind / apply / call

The purpose is to change the “this” referring.

  • bind takes one or more arguments, the first would be the new “this” value -> const newFunc = func.bind(antherObj);
  • “apply” method calls a function with a specified “this” value and arguments provided as an array. The first argument to the “apply” method is the new “this” value, and the second argument is an array of arguments to be passed to the function. -> apply(anotherObj, [arg1, arg2])
  • “call” method is similar to the “apply” method, but the arguments are passed individually rather than as an array. The first argument to the “call” method is the new “this” value, and the remaining arguments are individual arguments to be passed to the function.

-> call(anotherObj, arg1, arg2)

Event bubbling / Event capturing

bubbling

  • mechanism in JS that allows an event to propagate through the DOM hierarchy -> from the innermost element to the outermost element.
  • When an event occurs on an element, such as a click or a keypress, the event is first handled by the element itself. Then, the event “bubbles up” to its parent element, and so on, until it reaches the top-level document object.
  • Allows us to capture an event on a parent element rather than handling it on every child element individually. This can be useful when working with dynamic HTML content or when we want to minimize the number of event listeners in our code.

capturing

Event capturing is another mechanism in JavaScript that allows an event to be captured and handled by a parent element before it reaches the target element. In contrast to event bubbling, event capturing starts from the outermost parent element and propagates towards the target element.

e.target / e.currentTarget

e.target: The thing under the mouse (the thing that triggers the event).
e.currentTarget is the element with the added event listener.

Event delegation

Event Delegation is the method. this method will let you create one event listener to trigger all child elements. Each child element will be targeted by a unique ID.

JS prototype

  1. Why do we need the “prototype”? To re-use method in the object
  2. Each function automatically has a “prototype” property. The value of “prototype” is an object.
  3. Each object(instance) has a “__proto__” property. The “__proto__” refers the “prototype” of its creator(the function).
  4. You can use a function(constructor) with “new” keyword to create an object(instance). Then the created object’s “__proto__” refers the “prototype” of the function(constructor)
  5. the object will have a “constructor” property automatically, which refers to the constructor(the function)
  6. Arrow function cannot be a constructor
  7. When accessing a property of an object, JS will look at itself first. If no, it will look at the “proto__”, If still no, it will look at “__proto.__proto__” until the “__proto__” is null. (the prototype chain)

Why put the tag in the last of HTML file?

Putting the script tag in the last can reduce the delay time, users won’t wait a long time to open the webpage. Because the render direction is from top to bottom.

IIFE (Immediately Invoked Function Expression)

IIFE stands for Immediately Invoked Function Expression. It is a JavaScript design pattern that involves defining and immediately invoking a function, usually for the purpose of creating a private scope for variables and functions.
